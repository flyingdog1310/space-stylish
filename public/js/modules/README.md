# 前端模組化結構說明

## 目錄結構
```
public/
├── js/
│   ├── modules/           # 功能模組
│   │   ├── auth/         # 認證相關
│   │   ├── product/      # 產品相關
│   │   ├── cart/         # 購物車相關
│   │   ├── order/        # 訂單相關
│   │   ├── user/         # 用戶相關
│   │   ├── admin/        # 管理員相關
│   │   └── marketing/    # 行銷相關
│   ├── components/       # 可重用組件
│   ├── utils/           # 工具函數
│   ├── api/             # API 調用
│   └── pages/           # 頁面特定腳本
├── css/
│   ├── modules/         # 模組化樣式
│   ├── components/      # 組件樣式
│   └── pages/          # 頁面特定樣式
└── assets/             # 靜態資源
    ├── images/
    └── icons/
```

## 模組化原則
1. **按功能分組**: 每個業務模組有自己的目錄
2. **組件化**: 可重用的UI組件獨立管理
3. **工具函數**: 通用工具函數集中管理
4. **樣式模組化**: CSS按功能模組組織
5. **資源管理**: 靜態資源統一管理
